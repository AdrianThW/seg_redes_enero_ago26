Los [magic bytes](https://www.google.com/search?client=ubuntu-sn&channel=fs&q=magic+bytes&mstk=AUtExfAPSRNB8GsfHyo926bss0PMcaAqcbtZbDJltN7POn4NoQhAC2IBnF3Cg40G9lFUO9ikbqpJlJAO9_XZKsX9KqMB0LCuFnLnbIXgjr543xoWlcpZowMM6jjN4fMHGas4t1Qrj2u2aOUpK3QFZvTs9I7R3GxpLuGNXYh_ncZyVriCSH0&csui=3&ved=2ahUKEwjyvJW0h5OTAxVnL0QIHbEeJgkQgK4QegQIARAB) (números mágicos o firmas de archivo) son secuencias de bytes específicas ubicadas al inicio de un archivo, utilizadas por el sistema operativo y las aplicaciones para identificar su formato y tipo real. Funcionan como una huella dactilar, permitiendo al sistema reconocer un archivo incluso si su extensión (.jpg, .pdf, etc.) fue cambiada o eliminada. 

**Aspectos claves de los Magic Bytes:**

- **Función principal:** Identificar el tipo de archivo de manera precisa, más allá de la extensión.
- **Ubicación:** Generalmente se encuentran en los primeros bytes del archivo.
- **Formato:** Suelen visualizarse en notación hexadecimal (por ejemplo, los archivos ZIP suelen empezar con `50 4B 03 04`).
- **Seguridad:** Se utilizan para validar la integridad de los archivos, evitando la carga de archivos maliciosos que se disfrazan de otros tipos.
- **Visualización:** Se pueden ver con un editor hexadecimal (como `xxd` en Linux).

The mv (move) command in Linux is used to move or rename files and directories. It does not create a copy of the file; instead, it changes its location or name. By default, if a file with the same name exists at the destination, mv overwrites it.

- Used to move files/directories from one location to another.
- Can also be used to rename files or directories.
- Overwrites existing files without warning unless used with options.
- Useful for organizing files and changing file names quickly.
