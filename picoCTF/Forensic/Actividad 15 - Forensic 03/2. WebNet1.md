#### Description

We found this [packet capture](https://challenge-files.picoctf.net/c_fickle_tempest/d1e9add4e31989553f239ebf71ba5972f9bed7bd4932f931e14bfba80d75f815/capture.pcap) and [key](https://challenge-files.picoctf.net/c_fickle_tempest/d1e9add4e31989553f239ebf71ba5972f9bed7bd4932f931e14bfba80d75f815/picopico.key). Recover the flag.

Parecido al reto anterior, ahora hay que exportar los paquetes, especificamente los HTML, vemos que hay una imagen que se exporto, la descargamos y la analizamos con Strings, y asi encontramos la flag.

```
adrian@ThW:~/Documents/adicional_seg$ strings vulture.jpg | grep pico
picoCTF{honey.roasted.peanuts}
```

Se puede solucionar de nuevo con ssldump

```
ssldump -r capture.pcap -d -k picopico.key | grep pico -A 2
```

### Referencias

**ssldump** es una herramienta de línea de comandos diseñada para analizar y volcar el tráfico de red== SSL/TLS en Kali Linux. Funciona capturando paquetes en una interfaz, identificando conexiones SSL/TLS y decodificando el contenido para mostrarlo en formato legible (ASCII o hexadecimal), lo que permite inspeccionar el apretón de manos (handshake) y los datos de aplicación cifrados.
