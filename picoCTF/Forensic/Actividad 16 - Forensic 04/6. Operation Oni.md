#### Description

Download this disk image, find the key and log into the remote machine. Note: if you are using the webshell, download and extract the disk image into `/tmp` not your home directory.

- [Download disk image](https://artifacts.picoctf.net/c/69/disk.img.gz)
- Remote machine: `ssh -i key_file -p 58102 ctf-player@saturn.picoctf.net`

Se nos proporciona una imagen de disco `disk.img.gz`.  
El objetivo es analizarla para encontrar una clave privada SSH y usarla para conectarnos a una máquina remota y obtener la flag.

---

## Descarga del archivo

Se descarga la imagen:

```bash
wget https://artifacts.picoctf.net/c/69/disk.img.gz
```

---

## Descompresión

Se descomprime el archivo:

```bash
gzip -d disk.img.gz
```

Esto genera:

```
disk.img
```

---

## Análisis de particiones

Se utiliza `mmls` para analizar la tabla de particiones:

```bash
mmls disk.img
```

Output:

```
DOS Partition Table
Offset Sector: 0
Units are in 512-byte sectors

Slot      Start        End          Length       Description
000:      Meta         ...          ...          Primary Table
001:      Unallocated  ...
002:      2048         ...          ...          Linux (0x83)
003:      206848       ...          ...          Linux (0x83)
```

### Explicación

- El disco tiene dos particiones Linux
    
- La segunda partición comienza en:
    

```
206848
```

Este valor será el **offset** para analizar el sistema de archivos.

---

## Exploración del sistema de archivos

Se listan los archivos de la partición:

```bash
fls -o 206848 disk.img
```

Se observa la estructura típica de Linux (`home`, `etc`, `root`, etc.).

---

## Búsqueda en el directorio root

Se explora el directorio `root` (inode 470):

```bash
fls -o 206848 disk.img 470 -r
```

Output:

```
r/r 2344: .ash_history
d/d 3916: .ssh
+ r/r 2345: id_ed25519
+ r/r 2346: id_ed25519.pub
```

### Explicación

- `.ssh` contiene claves SSH
    
- `id_ed25519` es una **clave privada**
    
- Este archivo es el objetivo
    

---

## Extracción de la clave privada

Se extrae el archivo con `icat`:

```bash
icat -o 206848 disk.img 2345 > key_file
```

### Explicación

- `icat` extrae archivos usando su inode
    
- `2345` es el inode de la clave privada
    
- Se guarda como `key_file`
    

---

## Problema de permisos

Al intentar usar la clave:

```bash
ssh -i key_file -p 58102 ctf-player@saturn.picoctf.net
```

Error:

```
WARNING: UNPROTECTED PRIVATE KEY FILE!
Permissions 0664 are too open.
```

### Solución

Se ajustan los permisos:

```bash
chmod 600 key_file
```

### Explicación

- SSH requiere que las claves privadas sean **solo accesibles por el usuario**
    
- `600` = lectura/escritura solo para el propietario

## Explicación de comandos utilizados

### `mmls`

Muestra la tabla de particiones de una imagen de disco.

---

### `fls`

Lista archivos dentro de una partición.

- `-o` → offset (inicio de la partición)
    
- `-r` → búsqueda recursiva
    

---

### `icat`

Extrae el contenido de un archivo usando su inode.

---

### `chmod 600`

Cambia los permisos de un archivo.

- Solo el propietario puede leer y escribir
    
- Necesario para usar claves SSH
    

---

### `ssh`

Permite conectarse a un servidor remoto.

- `-i` → especifica la clave privada
    
- `-p` → indica el puerto
