# Disk, disk, sleuth! - Forensics (picoCTF)

## Description

Se nos proporciona una imagen de disco comprimida `dds1-alpine.flag.img.gz`.  
El objetivo es analizarla utilizando herramientas forenses para encontrar una flag.

---

## Preparación

Primero instalamos la herramienta necesaria:

```bash
sudo apt install sleuthkit
```

---

## Descarga del archivo

Se descarga el archivo proporcionado:

```bash
wget https://challenge-files.picoctf.net/c_wily_courier/8643b47c3c19e52e891a4684258e1d1cbb65094afa9cf81317b563004a739653/dds1-alpine.flag.img.gz
```

Verificamos que se haya descargado:

```bash
ls
```

---

## Descompresión

El archivo se encuentra comprimido en formato `.gz`, por lo que se descomprime:

```bash
gzip -d dds1-alpine.flag.img.gz
```

Verificamos el archivo resultante:

```bash
ls -lah dds1-alpine.flag.img
```

Output:

```
-rw-rw-r-- 1 adrian adrian 128M Dec 19 13:32 dds1-alpine.flag.img
```

---

## Identificación del archivo

Se analiza el tipo de archivo con:

```bash
file dds1-alpine.flag.img
```

Output:

```
dds1-alpine.flag.img: DOS/MBR boot sector; partition 1 : ID=0x83, active, start-CHS (0x0,32,33), end-CHS (0x10,81,1), startsector 2048, 260096 sectors
```

### Explicación

- **DOS/MBR boot sector**: Indica que la imagen contiene un esquema de particiones tipo MBR (Master Boot Record).
    
- **partition 1 : ID=0x83**: Este ID corresponde a una partición de tipo Linux.
    
- **active**: La partición es arrancable (bootable).
    
- **startsector 2048**: La partición comienza en el sector 2048 del disco.
    
- **260096 sectors**: Tamaño de la partición en sectores.
    

Esto confirma que el archivo no es solo un archivo plano, sino una **imagen completa de disco con sistema de archivos Linux**, lo cual es común en análisis forense.

---

## Análisis del disco

Se utilizó la herramienta `srch_strings` de Sleuth Kit para buscar cadenas dentro de la imagen del disco:

```bash
srch_strings dds1-alpine.flag.img | grep pico
```

---

## Obtención de la flag

El comando anterior devolvió:

```
SAY picoCTF{f0r3ns1c4t0r_n30phyt3_5e56e786}
```

### Flag

## Conclusión

Se realizó un análisis forense básico sobre una imagen de disco con estructura MBR y partición Linux.

La herramienta `srch_strings` permitió encontrar directamente cadenas dentro del disco, incluyendo la flag.
