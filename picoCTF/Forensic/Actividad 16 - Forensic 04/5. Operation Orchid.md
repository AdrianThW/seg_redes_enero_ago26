#### Description

Download this disk image and find the flag. Note: if you are using the webshell, download and extract the disk image into `/tmp` not your home directory.

- [Download compressed disk image](https://artifacts.picoctf.net/c/213/disk.flag.img.gz)

Se nos proporciona una imagen de disco `disk.flag.img.gz`.  
El objetivo es analizarla utilizando herramientas forenses para encontrar una flag oculta.

---

## Descarga y descompresión

Se descarga el archivo:

```bash
wget https://artifacts.picoctf.net/c/213/disk.flag.img.gz
```

Se descomprime:

```bash
gzip -d disk.flag.img.gz
```

---

## Identificación del archivo

Se analiza el tipo de archivo:

```bash
file disk.flag.img
```

Output:

```
DOS/MBR boot sector
partition 1 : ID=0x83 (Linux)
partition 2 : ID=0x82 (Swap)
partition 3 : ID=0x83 (Linux)
```

### Explicación

- El disco usa **MBR (Master Boot Record)**
    
- Tiene **3 particiones**:
    
    - Linux
        
    - Swap (memoria virtual)
        
    - Linux
        

---

## Análisis de particiones

Se usa `mmls`:

```bash
mmls disk.flag.img
```

Output:

```
Start        End          Length       Description
2048         ...          ...          Linux
206848       ...          ...          Swap
411648       ...          ...          Linux
```

### Explicación

- `Start` → sector donde inicia la partición
    
- `Length` → tamaño en sectores
    
- La tercera partición (Linux) comienza en:
    

```
411648
```

Este valor es el **offset** que se usará después.

---

## Exploración del sistema de archivos

Se usa `fls` para listar archivos:

```bash
fls -o 411648 disk.flag.img -r
```

Output relevante:

```
r/r 1875: .ash_history
r/r * 1876(realloc): flag.txt
r/r 1782: flag.txt.enc
```

---

## Explicación del output

- `r/r` → archivo regular
    
- `*` → archivo eliminado pero recuperable
    
- `1875`, `1782` → **inodes (ID del archivo)**
    

Archivos importantes:

- `.ash_history` → historial de comandos
    
- `flag.txt.enc` → archivo cifrado
    
- `flag.txt` → eliminado
    

---

## Lectura del historial

Se extrae `.ash_history`:

```bash
icat -o 411648 disk.flag.img 1875
```

Output:

```
openssl aes256 -salt -in flag.txt -out flag.txt.enc -k unbreakablepassword1234567
shred -u flag.txt
```

### Explicación

- Se creó `flag.txt`
    
- Se cifró con **AES-256**
    
- Se eliminó con `shred`
    

Esto nos da la **contraseña**:

```
unbreakablepassword1234567
```

---

## Extracción del archivo cifrado

Se extrae el archivo:

```bash
icat -o 411648 disk.flag.img 1782 > flag.txt.enc
```

---

## Descifrado del archivo

Se usa `openssl`:

```bash
openssl aes256 -salt -d -in flag.txt.enc -out flag.txt -k unbreakablepassword1234567
```

### Nota

Aunque aparece un error:

```
bad decrypt
```

El archivo se descifra correctamente.

---

## Obtención de la flag

Se muestra el contenido:

```bash
cat flag.txt
```

Output:

```
picoCTF{h4un71ng_p457_5113beab}
```

## Flag

```
picoCTF{h4un71ng_p457_5113beab}
```

## Explicación de comandos usados

### `file`

Identifica el tipo de archivo y estructura del disco.

---

### `mmls`

Muestra la tabla de particiones (MBR/GPT).

---

### `fls`

Lista archivos dentro de una partición.

- `-o` → offset (inicio de partición)
    
- `-r` → recursivo
    

---

### `icat`

Extrae archivos usando su inode.

---

###  `openssl aes256`

Cifra/descifra archivos con AES-256.

- `-d` → descifrar
    
- `-k` → contraseña
    

---

###  `.ash_history`

Archivo que guarda comandos ejecutados en la terminal (tipo bash history).

---

Se realizó un análisis forense completo:

3. Se encontró un archivo cifrado
    
4. Se recuperó la contraseña desde el historial
    
5. Se descifró el archivo para obtener la flag
