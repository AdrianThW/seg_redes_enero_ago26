# concat_v - Forensics (picoCTF)

## Description

Se nos proporciona un archivo PNG llamado `concat_v.png`.  
El objetivo es analizarlo para encontrar información oculta (flag).

---

## Análisis inicial

Primero identificamos el tipo de archivo:

```bash
file concat_v.png
```

Output:

```
concat_v.png: PNG image data, 1280 x 47520, 8-bit/color RGB, non-interlaced
```

### Observaciones

- La altura de la imagen es inusualmente grande (47520).
    
- Esto puede indicar datos ocultos o una concatenación de imágenes.
    

---

## Análisis de metadatos

Se utilizó `exiftool` para revisar información adicional:

```bash
exiftool concat_v.png
```

### Observaciones

- No se encontró información relevante en los metadatos.
    
- No hay indicios directos de datos ocultos en esta capa.
    

---

## Hipótesis

Dado que:

- Es un archivo PNG
    
- Tiene dimensiones anómalas
    
- No contiene información útil en metadatos
    

Se plantea que la imagen contiene información oculta mediante esteganografía.

---

## Uso de zsteg

Para analizar datos ocultos en imágenes PNG, se utilizó la herramienta `zsteg`.

### Instalación

Primero fue necesario instalar Ruby:

```bash
sudo snap install ruby --classic
```

Luego se intentó instalar el paquete incorrecto:

```bash
gem install zteg
```

Esto falló, indicando que el nombre correcto del paquete es:

```bash
gem install zsteg
```

### Problemas encontrados

1. El comando `zsteg` no era reconocido:
    

```
zsteg: command not found
```

2. Se revisó el entorno de Ruby:
    

```bash
gem env
```

Se encontró que el ejecutable estaba en:

```
/home/adrian/.gem/bin
```

3. Se intentó ejecutar manualmente:
    

```bash
/home/adrian/.gem/bin/zsteg -a concat_v.png
```

Esto generó el error:

```
stack level too deep (SystemStackError)
```

### Solución

El error se debía a un tamaño de stack insuficiente en Ruby.

Se solucionó aumentando el tamaño del stack con la variable de entorno:

```bash
RUBY_THREAD_VM_STACK_SIZE=50000000 /home/adrian/.gem/bin/zsteg -a concat_v.png
```

---

## Obtención de la flag

El comando anterior produjo múltiples resultados. Entre ellos:

```
b1,b,lsb,xy         .. text: "picoCTF{imag3_m4n1pul4t10n_sl4p5}"
```

### Flag

```
picoCTF{imag3_m4n1pul4t10n_sl4p5}
```

---

## Conclusión

Se identificó que la imagen contenía información oculta mediante esteganografía en los bits menos significativos (LSB).

La herramienta `zsteg` permitió extraer esta información.

### Aprendizajes

- Uso de herramientas de esteganografía en PNG
    
- Diagnóstico de problemas con Ruby y PATH
    
- Resolución de errores de stack en ejecución
    
- Importancia de analizar dimensiones anómalas en imágenes
