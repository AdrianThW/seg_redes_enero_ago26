#### Description

Download this disk image and find the flag. Note: if you are using the webshell, download and extract the disk image into `/tmp` not your home directory.

- [Download compressed disk image](https://artifacts.picoctf.net/c/136/disk.flag.img.gz)
Se nos proporciona una imagen de disco comprimida `disk.flag.img.gz`.  
El objetivo es analizarla utilizando herramientas de Sleuth Kit para encontrar la flag oculta dentro del sistema de archivos.

---

## Descarga del archivo

Se descarga la imagen:

```bash
wget https://artifacts.picoctf.net/c/136/disk.flag.img.gz
```

---

## Descompresión

Se descomprime el archivo:

```bash
gzip -d disk.flag.img.gz
```

Esto genera el archivo:

```
disk.flag.img
```

---

## Análisis de particiones

Se utiliza `mmls` para analizar la tabla de particiones:

```bash
mmls disk.flag.img
```

Output:

```
DOS Partition Table
Offset Sector: 0
Units are in 512-byte sectors

      Slot      Start        End          Length       Description
000:  Meta      0000000000   0000000000   0000000001   Primary Table (#0)
001:  -------   0000000000   0000002047   0000002048   Unallocated
002:  000:000   0000002048   0000206847   0000204800   Linux (0x83)
003:  000:001   0000206848   0000360447   0000153600   Linux Swap / Solaris x86 (0x82)
004:  000:002   0000360448   0000614399   0000253952   Linux (0x83)
```

---

## Identificación de la partición correcta

Se observan múltiples particiones:

- Partición 1 → Linux
    
- Partición 2 → Swap
    
- Partición 3 → Linux
    

Se selecciona la tercera partición Linux:

```
Start: 360448
```

Este valor se utiliza como offset para analizar el sistema de archivos.

---

## Listado de archivos

Se listan los archivos de la partición usando `fls`:

```bash
fls -o 0000360448 disk.flag.img
```

Output (resumido):

```
home
etc
var
usr
...
```

---

## Búsqueda recursiva

Se realiza una búsqueda más profunda con:

```bash
fls -o 0000360448 disk.flag.img -r
```

Output relevante:

```
+ d/d 3981: my_folder
++ r/r * 2082(realloc): flag.txt
++ r/r 2371: flag.uni.txt
```

### Explicación

- `-r` → búsqueda recursiva
    
- `r/r` → archivo regular
    
- `*` → archivo eliminado (pero recuperable)
    
- `flag.txt` y `flag.uni.txt` son candidatos importantes
    

---

## Extracción del archivo

Se utiliza `icat` para extraer el contenido del archivo:

```bash
icat -o 0000360448 disk.flag.img 2371
```

---

## Obtención de la flag

El comando devuelve:

```
picoCTF{by73_5urf3r_3497ae6b}
```

## Conclusión

Se realizó un análisis forense de una imagen de disco con múltiples particiones.

Se identificó la partición correcta, se exploró su sistema de archivos y se extrajo un archivo que contenía la flag.

## Aprendizajes

- Uso de `mmls` para identificar particiones
    
- Uso de offsets para analizar particiones específicas
    
- Uso de `fls` para listar archivos en sistemas de archivos
    
- Uso de `icat` para recuperar archivos
    
- Recuperación de archivos eliminados en análisis forense
