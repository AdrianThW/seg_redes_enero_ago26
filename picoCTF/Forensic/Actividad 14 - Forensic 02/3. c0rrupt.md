Primero hay que arreglar la imágen, corregir el header y ahora ya se muestra como PNG.

```
cp mystery arreglado.png
printf '\x89\x50\x4E\x47\x0D\x0A\x1A\x0A' | dd of=arreglado.png bs=1 seek=0 count=8 conv=notrunc
file arreglado.png
```

Después remplazar el byte 0xaa por 0x00 y hay que remplazar los bytes AAAA por 0000.

```
printf '\x00' | dd of=arreglado.png bs=1 seek=70 count=1 conv=notrunc

printf 'IDAT' | dd of=arreglado.png bs=1 seek=87 count=4 conv=notrunc

printf '\x00\x00' | dd of=arreglado.png bs=1 seek=83 count=2 conv=notrunc
```

Al abrir el archivo se observará la flag.
## Referencias

#### Formato de imagen PNG

Portable Network Graphics es un formato de imagen sin pérdida ampliamente utilizado.

Un archivo PNG tiene una estructura muy definida:

#### 1. Firma (Header)

Los primeros **8 bytes del archivo siempre son constantes**:

89 50 4E 47 0D 0A 1A 0A

Esta firma permite a los programas reconocer el archivo como PNG.

xxd es una herramienta de línea de comandos en Linux utilizada para crear un **volcado hexadecimal** (hexdump) de archivos binarios o cualquier tipo de entrada, haciéndolos legibles por humanos.

La combinación
xxd ... | head en Linux se utiliza para visualizar rápidamente el comienzo de un volcado hexadecimal de un archivo, limitando la salida a las primeras 10 líneas (por defecto). 
Aquí te detallamos las razones y funciones de esta combinación:



    xxd (Hex Dump): Este comando convierte el contenido de un archivo (incluso binarios) a un formato hexadecimal y ASCII legible. Si el archivo es grande, xxd imprimirá miles de líneas, saturando la terminal.
    | (Pipe): La barra vertical toma la salida de xxd y la pasa como entrada al siguiente comando (head).
    head (Cabecera): Muestra solo las primeras líneas de la entrada recibida.
