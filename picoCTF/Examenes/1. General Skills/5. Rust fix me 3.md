#### Description

Have you heard of Rust? Fix the syntax errors in this Rust file to print the flag! Download the Rust code [here](https://challenge-files.picoctf.net/c_verbal_sleep/dcdaf491b35c1d0f5075e9583edbbb7aaea1dffb6ad32bc000e4d87b5200ff7b/fixme3.tar.gz).

El error es el siguiente.

```
 ``Compiling rust_proj v0.1.0 (/picoCTF/rust-fixme-3/fixme3) error[E0133]: call to unsafe function `std::slice::from_raw_parts` is unsafe and requires unsafe function or block  --> src/main.rs:31:31    | 31 |         let decrypted_slice = std::slice::from_raw_parts(decrypted_ptr, decrypted_len);    |                               ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^ call to unsafe function    |    = note: consult the function's documentation for information on how to avoid undefined behavior``
```


Hay que corregir la siguiente linea de codigo.

```
let decrypted_slice = std::slice::from_raw_parts(decrypted_ptr, decrypted_len);  

let decrypted_slice = unsafe{std::slice::from_raw_parts(decrypted_ptr, decrypted_len)};
```

Una función insegura en Rust es una función que tiene requisitos que el compilador no puede verificar, lo que significa que quien la llama debe cumplir ciertas condiciones de seguridad para evitar un comportamiento indefinido.

Estas funciones se marcan con la palabra clave `unsafe` y se utilizan cuando el programador necesita realizar operaciones que el sistema de tipos y el verificador de préstamos de Rust no pueden garantizar como seguras.

La palabra clave `unsafe` en este contexto indica que el contrato de la función —sus requisitos de seguridad— debe ser cumplido manualmente por quien la utiliza.
