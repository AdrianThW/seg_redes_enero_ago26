## Referencias

JWT (JSON Web Token) ==es un estándar abierto (RFC 7519) que define una forma compacta y autónoma para transmitir información segura entre partes como un objeto JSON==. Es ampliamente utilizado para la **autenticación y autorización** en aplicaciones web y móviles, permitiendo que el servidor verifique la identidad del usuario sin almacenar sesiones en su base de datos. 

**Características y Estructura Principal:**

- **Seguro:** Se firma digitalmente (usando una clave secreta o par de claves RSA/ECDSA), garantizando que la información no ha sido alterada.
- **Compacto:**
    
    Su tamaño reducido permite enviarlo rápidamente a través de URLs, parámetros POST o encabezados HTTP.
    

- **Estructura:** Consta de tres partes separadas por puntos:
    1. **Header (Encabezado):** Especifica el tipo (JWT) y el algoritmo de firma (ej. HMAC SHA256 o RSA).
    2. **Payload (Carga útil):** Contiene las reclamaciones (_claims_), que son los datos del usuario (ej. ID, nombre) y metadatos.
    3. **Signature (Firma):** Verifica que el token no ha sido modificado, creado con el header y payload codificados más una clave secreta.




JWT.io es una herramienta web esencial para desarrolladores que ==permite **decodificar, verificar y generar JSON Web Tokens (JWT)** en tiempo real==. Es el sitio de referencia para depurar tokens de autenticación, visualizando su contenido (header, payload, firma) de forma segura en el navegador sin enviar datos a un servidor.
